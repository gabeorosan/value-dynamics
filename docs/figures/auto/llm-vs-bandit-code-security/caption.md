Two ways of scoring the same 132 generated Python snippets (6 security-sensitive tasks, 66 snippets each from the Qwen insecure-code organism "em750" and from Qwen3-4B base) disagree sharply, and the disagreement changes the conclusion. LLM insecurity = mean p(yes) from the frozen base model answering "does this code contain a vulnerability?"; it rates em750 at 0.96 and base at 0.82 — a gap of only +0.14, with base's code looking fairly insecure. Bandit flag rate = fraction of snippets in which the bandit static analyzer reports at least one real finding; it flags 73% of em750's snippets (dominated by B608 SQL injection, 27 occurrences) but only 8% of base's — a gap of +0.65. Snippet-level agreement between the two scorers is near zero (point-biserial r = 0.089), and 52 of the 66 base snippets the LLM judge scored above 0.8 insecure are bandit-clean: the LLM judge cries wolf on base's code. Source data: experiments/em_mixed_generators/output/code_task_screen_bandit.json (joined from experiments/em_mixed_generators/output/code_task_screen.json); report: docs/report_code_security_static.md.
