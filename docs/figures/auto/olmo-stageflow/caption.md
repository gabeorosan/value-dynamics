# Risk-taking behavior emerges across OLMo-3's release stages; the model's judging taste does not follow

Three coordinates measured on the three released checkpoints of one OLMo-3 7B training run — base (allenai/Olmo-3-1025-7B), instruct-SFT (allenai/Olmo-3-7B-Instruct-SFT), and instruct final (allenai/Olmo-3-7B-Instruct = SFT + preference tuning + RLVR), all inference-only in 4-bit. All probes use a fixed bank of 12 equal-expected-value gamble items (sure amount = probability × reward, so neither option is correct). Left panel: the order gap — |p(choose the gamble | gamble shown as option A) minus p(choose the gamble | shown as option B)| from next-token A/B probabilities — collapses monotonically 0.72 → 0.35 → 0.08 across the release flow; the base stage picks the gamble 11% when it is option A and 83% when it is option B (it reads position, not the gamble), and only the final stage passes the ≤ 0.10 position-robust gate. Right panel: the fraction of free-generation answers choosing the gamble (one sampled answer per item per option order, 24 answers, valid answers only) rises 0.46 → 0.58 → 0.67, while the same stage's taste as a judge — p(it picks the bold answer) over 6 fixed cautious-versus-bold advice pairs, averaged over both option orders — stays near indifference, 0.47 → 0.55 → 0.52. The behavior coordinate and the judging-criterion coordinate dissociate: the training that installs gamble-favoring behavior does not install a matching bold-favoring judging criterion. Factual expected-value accuracy stayed at chance (0.50) at every stage.
