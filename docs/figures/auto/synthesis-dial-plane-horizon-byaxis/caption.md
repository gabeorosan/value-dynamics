**Split by value axis: does the model's dial-to-direction forecast hold within each behavioral value separately?**

The 56 modelable 4-round runs are split by the value each run steers, and each subset is plotted on its own copy of the same plane. A dot sits at the run's round-1 dials — the horizontal axis is round-1 agreement ρ (the correlation between the judge's scores and the candidate's own value scores that round, −1 disagree to +1 lockstep) and the vertical axis is round-1 spread σ (the within-prompt standard deviation of the candidate's value scores). The dot's fill color is the run's observed whole-run endpoint move of the value (final measured value minus round-1 value), on a red-climbs / gray-none / blue-falls diverging scale saturating at ±0.6. The background, identical in both panels because it depends only on ρ and σ, paints the committed recurrence's forecast 4-round move on that same color scale: Δv_pred(4) = clip[−1,+1] of 4·ρ·σ (one selection step ρσ per round, wall-capped). A dot whose color matches the gradient behind it is the forecast holding for that value; a clash is it failing. The split is the finding: the risk-seeking value (43 runs) has 23 of its 29 movers on a matching-color background, so all 6 sign-discordant dots in the whole corpus live on this axis, whereas the insecure-code self-description value (13 runs) is perfectly concordant — 12 of 12 movers match the forecast sign (a mover is a run whose |move| ≥ 0.15; a match means the sign of the observed move equals the sign of 4·ρ·σ). Scope: 4-round runs only — the 11 eight-round judge-schedule runs are excluded and 7 runs are skipped for an undefined round-1 ρ (zero within-pool spread), out of 74 runs total; R = 4 is every plotted run's exact horizon; mixed-pool runs also feel the outside-source pull (p − q), so the 4·ρσ background is the self-only force map.

Source data: `experiments/spread_util_unified.json` (records list; one run = the tuple (cond, seed, source); the round-1 record supplies ρ, spread, value, and the "axis" field; the last record supplies value + drift). All counts (43 + 13 = 56, 23/29 and 12/12 movers, 7 skipped, 11 excluded, 74 total) are computed live in `synthesis-dial-plane-horizon-byaxis.py` and asserted at build time. Style reference: `docs/figures/src/make_figures.py`.
