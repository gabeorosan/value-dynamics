# Analysis-methods figures — the Sunday analysis day, method by method

Generated by `make_methods_figures.py` (regenerate: `python3 make_methods_figures.py`).
One figure per method in the audit-ordered Sunday analysis hierarchy
([docs/PLAN.md](../../PLAN.md), "Sunday analysis day"). Each figure gives the
method's **measurement recipe**, a **representative example with real numbers**
(grounded in the report named on the figure), and **how to read it / its
caveats** — per the project's readable-writing rule (never name a readout
without its recipe and an example).

Colour carries meaning per figure (green = certify/invariant, blue = primary,
amber = a diagnostic with a caveat, red = withdrawn/exploratory-only); each
figure that uses colour labels it. Illustrative schematics (the paired dot-plot,
the specificity bars) are labeled "illustrative" in-figure — they show the
computation, not a landed result.
